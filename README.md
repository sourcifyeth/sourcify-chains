# sourcify-chains

Canonical chain configuration for [Sourcify](https://sourcify.dev) — the open-source smart contract verification service. The Sourcify server fetches [`sourcify-chains-default.json`](./sourcify-chains-default.json) from this repository at startup.

## Files

### `chain-overrides.json`

The manually maintained source of truth. Each entry defines a chain that Sourcify supports for verification:

```json
{
  "1": {
    "sourcifyName": "Ethereum Mainnet",
    "supported": true,
    "etherscanApi": {
      "supported": true,
      "apiKeyEnvName": "ETHERSCAN_API_KEY"
    },
    "fetchContractCreationTxUsing": {
      "etherscanApi": true,
      "blockscoutApi": { "url": "https://eth.blockscout.com/" }
    },
    "rpc": [
      {
        "type": "APIKeyRPC",
        "url": "https://eth-mainnet.g.alchemy.com/v2/{API_KEY}",
        "apiKeyEnvName": "ALCHEMY_API_KEY"
      }
    ]
  }
}
```

Full field reference: [services/server/README.md — Chains Config](https://github.com/sourcifyeth/sourcify/blob/staging/services/server/README.md#chains-config) in the main Sourcify repo.

### `sourcify-chains-default.json`

The published output fetched by the Sourcify server at startup via `config.chains.remoteUrl`. Currently identical to `chain-overrides.json`. In the future, this file will be auto-generated by merging `chain-overrides.json` with RPC endpoints from provider APIs (QuickNode, dRPC) and block explorer APIs (Etherscan, Blockscout, Routescan).

**Do not edit this file directly** — changes should go into `chain-overrides.json`.

## How to add a chain

1. Open a PR adding an entry to `chain-overrides.json`
2. At minimum, include `sourcifyName` and `supported`
3. Optionally add `rpc`, `etherscanApi`, and `fetchContractCreationTxUsing`
4. Once merged, `sourcify-chains-default.json` is updated and the Sourcify server will pick it up on the next restart
